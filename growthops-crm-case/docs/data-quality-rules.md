# Data Quality Rules

## 데이터 품질 이슈가 캠페인 성과를 왜곡하는 이유

캠페인 성과 지표는 리드 수, 유효 리드 수, 전환 수, 전환율을 기반으로 계산된다. 데이터 품질 이슈를 무시하면 실제보다 캠페인이 잘 작동하는 것처럼 보이거나, 반대로 좋은 캠페인이 낮게 평가될 수 있다.

예를 들어 중복 리드는 리드 수를 부풀리고, 이메일 누락 리드는 실제 후속 연락이 불가능한데도 리드로 집계된다. 존재하지 않는 캠페인 참조는 성과 귀속을 어렵게 만들고, 봇성 행동은 실제 고객 관심보다 행동 점수를 과도하게 높일 수 있다. 따라서 품질 검증은 스코어링 전 단계에서 반드시 수행해야 한다.

## HIGH / MEDIUM / LOW severity 기준

- `HIGH`: 자동 스코어링이나 캠페인 귀속 결과를 직접 왜곡할 수 있어 즉시 확인이 필요한 이슈
- `MEDIUM`: 반드시 오류라고 단정할 수는 없지만 캠페인 해석과 운영 판단에 영향을 줄 수 있는 이슈
- `LOW`: 보조 정보 누락처럼 핵심 식별자는 유지되지만 점수 계산 근거가 줄어드는 이슈

## 각 이슈 유형 설명

- `missing_email`: 리드 이메일이 비어 있다. 실제 연락이 어렵기 때문에 HIGH로 분류한다.
- `invalid_lead_campaign_reference`: 리드의 `source_campaign_id`가 캠페인 목록에 없다. 리드 유입 캠페인을 신뢰할 수 없어 HIGH로 분류한다.
- `invalid_event_lead_reference`: 이벤트의 `lead_id`가 리드 목록에 없다. 행동 데이터가 어떤 리드의 행동인지 알 수 없어 HIGH로 분류한다.
- `invalid_event_campaign_reference`: 이벤트의 `campaign_id`가 캠페인 목록에 없다. 이벤트를 캠페인 성과에 정확히 연결하기 어려워 HIGH로 분류한다.
- `invalid_conversion_lead_reference`: 전환 데이터의 `lead_id`가 리드 목록에 없다. 전환을 실제 리드에 연결할 수 없어 HIGH로 분류한다.
- `duplicate_email`: 같은 이메일이 여러 리드에 존재한다. 중복 후속 연락과 리드 수 과대 계산 위험이 있어 HIGH로 분류한다.
- `suspicious_bot_behavior`: 짧은 시간에 과도한 이벤트가 발생했다. 실제 고객 관심이 아닐 수 있어 HIGH로 분류한다.
- `duplicate_company_name`: 같은 회사명이 여러 리드에 존재한다. 같은 회사의 여러 담당자일 수도 있으므로 MEDIUM으로 분류한다.
- `missing_campaign_utm_code`: 캠페인의 UTM 코드가 비어 있다. 추적 품질은 낮아지지만 리드 식별 자체가 깨지는 것은 아니므로 MEDIUM으로 분류한다.
- `unknown_source_or_blank_optional_field`: 산업군, 회사 규모, 직무 등 보조 필드가 비어 있다. 점수 계산 근거가 줄어들기 때문에 LOW로 분류한다.

## Manual Review 분리 기준

다음 중 하나라도 해당하는 리드는 자동 점수 계산에서 제외하고 `Manual Review`로 분리한다.

- 이메일이 비어 있음
- 동일 이메일이 중복됨
- `source_campaign_id`가 `campaigns.csv`에 존재하지 않음
- 봇성 행동 의심으로 판단됨

Manual Review 분리는 리드를 삭제하는 처리가 아니다. 원천 행은 유지하되, 자동 영업 우선순위 계산 대상에서 제외해 운영자가 직접 확인할 수 있게 한다.

## 봇성 행동 의심 기준

동일 `lead_id`가 10분 이내 15개 이상의 이벤트를 발생시키면 `suspicious_bot_behavior`로 판단한다. 이 기준은 사람이 자연스럽게 수행하기 어려운 수준의 반복 행동을 탐지하기 위한 단순 규칙이다.

이번 더미데이터에서는 `L011`이 짧은 시간 안에 16개의 이벤트를 발생시키도록 구성되어 있으며, 자동 점수 계산에서 제외된다.

## 향후 실제 운영에서 추가할 수 있는 검증 규칙

- 이메일 형식 검증 및 일회용 이메일 도메인 탐지
- 회사 도메인과 이메일 도메인 일치 여부 확인
- `campaign_id`, `lead_id`, `event_id`, `conversion_id` 중복 여부 검증
- 캠페인 시작일과 종료일 범위를 벗어난 이벤트 탐지
- 전환 금액이 음수이거나 비정상적으로 큰 경우 탐지
- 이벤트 타입과 전환 타입의 허용 목록 검증
- 개인정보 수집 동의 여부 또는 수신 거부 상태 검증
- 같은 리드가 여러 캠페인에 동시에 귀속되는 경우의 어트리뷰션 규칙 적용
